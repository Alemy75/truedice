## 🧩 Раздел 5: HARDEST UNKNOWN (3:15 — 4:30) — **75 секунд** ⭐

**Это центральный пункт по правилам конкурса. Не торопись.**

**Говоришь:**

> "Самое интересное что я не знал — **разница между Chainlink VRF v2 и v2.5 на Sepolia**.
>
> Когда искал референсы и читал документацию, **80% туториалов всё ещё на v2**. Но Chainlink на Sepolia уже катит v2.5 как production. И между ними **breaking change в consumer API**:
>
> - Старая база: `VRFConsumerBaseV2`
> - Новая: `VRFConsumerBaseV2Plus`
> - Новый struct `RandomWordsRequest` с полем `extraArgs` для выбора **native ETH или LINK** оплаты
> - Другой coordinator address, другой key hash
>
> Я сначала задеплоил контракт с v2-импортами против v2.5-координатора. Транзакция `placeBet` проходила, **но fulfillment никогда не приходил**. Каждый раз получал ошибку **`request not paid`**.
>
> *Полтора часа* я смотрел не туда — думал что у меня LINK закончился в подписке, или что callback gas limit маленький. Спасло чтение **changelog'a Chainlink contracts npm пакета** — там в одной строке было написано про переход v2 → v2.5.
>
> Решение — запинил всё к v2.5:
> - `chainlink-brownie-contracts@1.3.0`
> - Sepolia v2.5 coordinator: `0x9DdfaCa8...`
> - Импорты `VRFConsumerBaseV2Plus` вместо `VRFConsumerBaseV2`
>
> **Бонус-quirk Sepolia**: после починки я ловил **timeouts на 30+ минут** — оказалось, Sepolia's LINK/ETH price feed настолько сломан что coordinator оценивает 200k gas callback в **40+ LINK** во время gas spikes. Mainnet equivalent — 2 LINK за тот же вызов. Документировал это в README как known testnet quirk.
>
> **Урок для меня**: на ранних версиях библиотек всегда проверять changelog, а не доверять туториалам — даже у такого крупного проекта как Chainlink **breaking changes между minor версиями есть**."

**Курсор**: если успеешь — открой `CasinoDice.sol` в VS Code, покажи импорт `VRFConsumerBaseV2Plus` и conструктор с координатором.

## 📐 Поток валюты — для уверенности (если жюри спросит)

```
1. Wallet ──ETH──> Contract.balanceOf[player]    (deposit, явный трансфер)
2. placeBet:
   balanceOf[player] -= stake                     (внутреннее списание)
   Contract ──request──> VRF Coordinator
   VRF Coord ──LINK──> Oracle                     (КАЗИНО платит, не игрок)
3. Oracle ──random + proof──> VRF Coord ──> Contract.fulfillRandomWords()
   ├── WIN:  houseBankroll −(payout − stake)──> balanceOf[player]
   └── LOSS: balanceOf[player] +stake уже списан, переезжает в houseBankroll
4. Wallet <──ETH── Contract                       (withdraw, явный трансфер)
```

**Ключевая мысль**: ETH физически переходит границу контракта **только при deposit и withdraw**. Между ставками — это просто числа в `mapping(address => uint256)` storage.
